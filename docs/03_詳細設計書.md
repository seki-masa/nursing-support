**状態・データ取得**:
- マウント時に対象者詳細（`/api/care-recipients/:id`）と全期間のバイタル（`/api/care-recipients/:id/vitals?days=all`）を並行取得する。最新バイタル表示と独立させるため取得は常に全期間で行う。

- ステータス（0〜6）・浮腫（0〜3）は数値レベルに変換して折れ線表示する。

### 1.4 VitalCard（バイタル値カード）

**責務**: 個別のバイタル値を表示し、異常値を視覚的にハイライトする

**異常値判定**:
```typescript
export const VITAL_RANGES: Record<string, { min: number; max: number; unit: string; label: string }> = {
  systolicBp:       { min: 90,   max: 139,  unit: 'mmHg', label: '収縮期血圧' },
  diastolicBp:      { min: 60,   max: 89,   unit: 'mmHg', label: '拡張期血圧' },
  heartRate:        { min: 60,   max: 100,  unit: 'bpm',  label: '心拍数' },
  respiratoryRate:  { min: 12,   max: 20,   unit: '回/分', label: '呼吸数' },
  temperature:      { min: 36.0, max: 37.4, unit: '℃',    label: '体温' },
  spo2:             { min: 96,   max: 100,  unit: '%',    label: 'SpO2' },
  bloodSugar:       { min: 70,   max: 140,  unit: 'mg/dL', label: '血糖値' },
  painScore:        { min: 0,    max: 10,   unit: '',     label: '疼痛スコア' },
}

export type AbnormalLevel = 'normal' | 'warning' | 'danger'

// 範囲外なら逸脱率15%超で danger、それ以下は warning
export function getAbnormalLevel(field: string, value: number): AbnormalLevel {
  const range = VITAL_RANGES[field]
  if (!range) return 'normal'
  if (value < range.min || value > range.max) {
    const deviation = Math.max(
      range.min > 0 ? (range.min - value) / range.min : 0,
      range.max > 0 ? (value - range.max) / range.max : 0,
    )
    return deviation > 0.15 ? 'danger' : 'warning'
  }
  return 'normal'
}

export function isAbnormal(field: string, value: number): boolean {
  return getAbnormalLevel(field, value) !== 'normal'
}
```

**カードスタイル**:
- `normal`: 白背景、グレーボーダー、「正常」表示
- `warning`: 黄背景、黄ボーダー、⚠ アイコン、「注意」表示
- `danger`: 赤背景、赤ボーダー、🔴 アイコン、「異常」表示

> 体重・疼痛スコア・浮腫は `VitalCard` を使わず、異常判定なしの無彩色カードで表示する（ダッシュボード側でインライン描画）。`painScore` は `VITAL_RANGES` に範囲を持つが、これは入力フォームのサーバ検証用で、カード表示の色分けには使わない。

**グラフ用レベル変換**（`lib/vitals-utils.ts` / `types/index.ts`）:

### 1.5 VitalInputForm（バイタル入力フォーム）

**バリデーションスキーマ（Zod）**:
```typescript
const vitalSchema = z.object({
  recordedAt: z.string().optional(),
  status: z.enum(['CRITICAL', 'SEVERE', 'CAUTION', 'OBSERVATION', 'HEALTHY', 'DECEASED', 'DISCHARGED']).optional().nullable(),
  deceasedAt: z.string().optional().nullable(),
  dischargedAt: z.string().optional().nullable(),
  systolicBp: z.number().int().min(40).max(300).optional().nullable(),
  diastolicBp: z.number().int().min(20).max(200).optional().nullable(),
  heartRate: z.number().int().min(20).max(300).optional().nullable(),
  respiratoryRate: z.number().int().min(0).max(60).optional().nullable(),
  temperature: z.number().min(30).max(45).optional().nullable(),
  spo2: z.number().int().min(50).max(100).optional().nullable(),
  weight: z.number().min(1).max(300).optional().nullable(),
  bloodSugar: z.number().int().min(20).max(600).optional().nullable(),
  consciousnessLevel: z.number().int().min(0).max(300).optional().nullable(),
  painScore: z.number().int().min(0).max(10).optional().nullable(),
  edema: z.enum(['NONE', 'MILD', 'MODERATE', 'SEVERE']).optional().nullable(),
  urineOutput: z.number().int().min(0).max(5000).optional().nullable(),
  notes: z.string().max(1000).optional().nullable(),
})
```

**UI仕様**:
- 記録日時はデフォルト現在日時（変更可能）
- ステータス（CareStatus）を記録時に選択可能。死亡/退院選択時は日時も記録する
- 「デバイスから取得」ボタン（将来実装、現状はdisabled + ツールチップ表示）
- 新規入力（`mode = 'create'`）と最新バイタル編集（`mode = 'edit'`、`vitalId` と初期値を受け取る）で同一フォームを共用する。

**入力バリデーション仕様**:
- ステータスが**死亡/退院以外**のとき、次の8項目は**必須**: 収縮期血圧・拡張期血圧・心拍数・呼吸数・体温・SpO2・体重・血糖値。死亡/退院時は必須を解除する。
- クライアントは文字列ベースのスキーマ（`superRefine`）で**必須未入力のみ**を検証する。数値の範囲は HTML の `min`/`max` を付けず（疼痛スコア・意識レベル・尿量含む）、ブラウザ標準検証で送信を妨げない。
- 値の範囲検証は**サーバ側**（`createSchema`/`updateSchema`）で行う。範囲外は 400 を返し、レスポンスの `error.fieldErrors` を各入力欄に `setError` で表示し、トーストでも通知する。
- 送信は `必須チェック → 値検証（サーバ）` の順。必須を満たさなければ送信前に止まる。

> 介護対象者の現在ステータスは別カラムではなく、最新バイタルの `status` から導出する。

### 1.5.1 最新バイタル編集

**責務**: 最新の1件のバイタルのみを再編集する

- 入口: ダッシュボード「最新バイタル」見出しの編集ボタン → `/care-recipients/[id]/vitals/[vitalId]/edit`。
- サーバコンポーネントで対象バイタルを取得し、**最新でなければ `notFound()`**。
- フォームは `VitalInputForm`（`mode='edit'`）を初期値付きで再利用し、保存は `PATCH /api/care-recipients/:id/vitals/:vitalId` を呼ぶ。
- API 側でも最新でない場合は 403（`code: NOT_LATEST`）を返す。

## 4. 認証設定

```typescript
// lib/auth.ts
import { NextAuthOptions } from 'next-auth'
import CredentialsProvider from 'next-auth/providers/credentials'
import bcrypt from 'bcryptjs'
import { prisma } from './prisma'

export const authOptions: NextAuthOptions = {
  providers: [
    CredentialsProvider({
      name: 'credentials',
      credentials: {
        email: { label: 'メールアドレス', type: 'email' },
        password: { label: 'パスワード', type: 'password' },
      },
      async authorize(credentials) {
        if (!credentials?.email || !credentials?.password) return null
        const user = await prisma.user.findUnique({
          where: { email: credentials.email },
        })
        if (!user) return null
        const valid = await bcrypt.compare(credentials.password, user.passwordHash)
        if (!valid) return null
        return { id: user.id, name: user.name, email: user.email, role: user.role, businessId: user.businessId }
      },
    }),
  ],
  callbacks: {
    jwt({ token, user }) {
      if (user) {
        token.id = user.id
        token.role = (user as any).role
        token.businessId = (user as any).businessId
      }
      return token
    },
    session({ session, token }) {
      if (session.user) {
        const u = session.user as { id?: string; role?: string; businessId?: string }
        u.id = token.id as string
        u.role = token.role as string
        u.businessId = token.businessId as string
      }
      return session
    },
  },
  pages: {
    signIn: '/login',
  },
  session: { strategy: 'jwt' },
}
```
