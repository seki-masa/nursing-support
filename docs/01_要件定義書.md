# 要件定義書

## 1. プロジェクト概要

### 1.1 システム名
介護支援バイタル管理システム（Nursing Vital Support System）

### 1.2 目的
介護施設における介護対象者のバイタルサインを一元管理し、異常値の早期発見と適切なケアの提供を支援するWebアプリケーションを構築する。

### 1.3 対象ユーザー
- 介護士・看護師（日常的なバイタル入力・閲覧）
- 施設管理者（対象者・スタッフ管理）
- 医師・医療スタッフ（記録の参照・判断）

---

## 2. 機能要件

### 2.1 介護対象者管理

#### 2.1.1 基本CRUD
- 新規登録
- 一覧表示（左サイドバー）
- 詳細表示（プロフィール画面）
- 編集
- 削除（論理削除）

#### 2.1.2 プロフィール項目

| 項目 | 型 | 必須 | 備考 |
|------|-----|------|------|
| 氏名 | テキスト | ○ | |
| 氏名（ふりがな） | テキスト | ○ | |
| 年齢 | 数値 | - | 誕生日から自動計算 |
| 性別 | 選択 | ○ | 男性/女性/その他 |
| 誕生日 | 日付 | ○ | |
| 血液型 | 選択 | - | A/B/O/AB（+/-） |
| 持病 | 複数テキスト | - | 複数登録可 |
| アレルギー | 複数テキスト | - | 複数登録可 |
| 病室 | テキスト | - | 例：301号室 |
| ステータス | 選択 | ○ | 下記参照 |
| 家族一覧 | 関連 | - | 多対多 |
| 備考 | テキスト（長文） | - | |

#### 2.1.3 ステータス定義（危険度順）

| ステータス | 表示色 | 危険度 |
|-----------|--------|--------|
| 危篤 | 赤 (#DC2626) | 最高 |
| 重篤 | オレンジ (#EA580C) | 高 |
| 要注意 | 黄 (#CA8A04) | 中 |
| 経過観察 | 青 (#2563EB) | 低 |
| 健康 | 緑 (#16A34A) | なし |
| 死亡 | グレー (#6B7280) | - |
| 退院 | グレー (#9CA3AF) | - |

### 2.2 家族管理

#### 2.2.1 家族プロフィール項目

| 項目 | 型 | 必須 |
|------|-----|------|
| 氏名 | テキスト | ○ |
| 続柄 | テキスト | ○ |
| 電話番号 | テキスト | - |
| メールアドレス | テキスト | - |
| 備考 | テキスト | - |

#### 2.2.2 多対多関係
- 1人の家族が複数の介護対象者に紐づけられる
- 1人の介護対象者に複数の家族を登録可能

### 2.3 バイタル管理

#### 2.3.1 バイタル項目

| 項目 | 単位 | 正常範囲（参考） | 備考 |
|------|------|----------------|------|
| 収縮期血圧（上） | mmHg | 90〜139 | |
| 拡張期血圧（下） | mmHg | 60〜89 | |
| 心拍数（脈拍） | bpm | 60〜100 | |
| 呼吸数 | 回/分 | 12〜20 | |
| 体温 | ℃ | 36.0〜37.4 | |
| SpO2（血中酸素飽和度） | % | 96〜100 | |
| 体重 | kg | - | 変動トレンドを監視 |
| 血糖値 | mg/dL | 70〜140（食後2h） | 任意 |
| 意識レベル | JCS | 0=清明 | Japan Coma Scale |
| 疼痛スコア | NRS | 0〜3（正常） | Numeric Rating Scale (0-10) |
| 浮腫 | 段階 | なし | なし/+/++/+++ |
| 尿量 | mL/日 | 1000〜2000 | 任意 |

#### 2.3.2 バイタル入力方法
1. **手動入力**: フォームから直接入力
2. **ウェアラブル連携（将来実装）**: Web Bluetooth APIを使用してBLE対応デバイスから自動取得

#### 2.3.3 バイタル表示
- 直近の値の一覧表示（カード形式）
- 時系列グラフ表示（折れ線グラフ、過去30日間）
- 異常値のハイライト表示（正常範囲外を赤色で警告）
- 記録日時・記録者の表示

### 2.4 認証・アクセス制御

| ロール | 権限 |
|--------|------|
| 管理者 (ADMIN) | 全機能 + ユーザー管理 |
| スタッフ (STAFF) | 閲覧・バイタル入力・プロフィール編集 |

## 3. 非機能要件

### 3.1 パフォーマンス
- ページ初期表示: 2秒以内
- バイタル保存: 1秒以内

### 3.2 セキュリティ
- ユーザー認証（メール/パスワード）
- ロールベースアクセス制御（管理者/スタッフ）
- 医療情報のHTTPS通信
- パスワードのbcryptハッシュ化

### 3.3 可用性
- レスポンシブデザイン（タブレット対応 768px以上）
- モダンブラウザ対応（Chrome, Firefox, Safari, Edge 最新2バージョン）

### 3.4 拡張性
- ウェアラブルデバイス連携（Web Bluetooth API）の将来実装を考慮したアーキテクチャ
- バイタル項目の追加が容易な設計

---

## 4. 制約条件

- PostgreSQLデータベースを使用
- UIは日本語
- Next.js App Routerアーキテクチャを採用
